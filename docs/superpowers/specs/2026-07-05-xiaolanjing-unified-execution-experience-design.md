# 小鲸鱼执行体验统一改造方案

日期：2026-07-05
状态：待审阅
目标：把小鲸鱼运行中的对话体验统一收口，重点对齐 Codex 的清爽过程流、输入框接管能力和完成后折叠体验。

## 一、核查范围

本方案基于以下内容判断：

- 小鲸鱼当前输入区、发送/停止按钮、附件区、运行中补充入口。
- 小鲸鱼当前过程展示、思考展示、命令展示、反问卡、步骤提示、完成折叠。
- 小鲸鱼当前任务记录方式，尤其是过程是否能持续保留。
- 既有方案：`docs/superpowers/specs/2026-07-04-xiaolanjing-codex-chat-iteration-design.md`
- 既有计划：`docs/superpowers/plans/2026-07-04-xiaolanjing-codex-chat-iteration.md`
- 执行体验说明：`projects/desktop-agent/docs/13-execution-interaction-prd.md`
- Codex、Claude、Marvis 三份桌面 Agent 逆向结论报告。
- 用户补充的 4 张 Codex 反问面板截图。

已覆盖：输入框、运行中补充、停止按钮、反问面板、步骤提示、思考摘要、过程时间线、命令折叠、完成折叠、颜色风格、异常白块问题入口。

尚未完全确认：运行命令时“小白块”出现的真实触发场景，需要后续实际跑任务复现后定位。

## 二、核心目标

这次不是单点修补，而是一轮体验统一改造。

用户发出任务后，小鲸鱼应该呈现为：

1. 状态清楚：一眼知道它在想、在做、还是等用户决定。
2. 过程连续：思考、动作和反馈像文档一样持续往下追加，不覆盖旧内容。
3. 默认安静：命令和细节不铺满屏幕，默认只露出必要一行。
4. 随时接管：输入框始终可用，停止、补充、反问选择都在输入区附近完成。
5. 完成利落：过程自动折叠，最终结果突出，摘要文案统一。
6. 色调统一：普通强调都用小鲸鱼蓝，只有错误和危险才用红色。

## 三、最终体验方案

### 3.1 发送按钮

运行中输入框为空时，右下角按钮变成“圆形内方块”的停止样式，不再使用叉号。

运行中输入框有内容时，按钮显示为追加发送，用户可以补充要求。补充内容进入当前任务的后续要求，不直接打断正在进行的动作。

验收：

- 运行中空输入：看到停止按钮。
- 运行中有输入：看到追加发送按钮。
- 点击停止后，任务结束并恢复普通输入状态。
- 点击追加发送后，输入框清空，过程继续。

### 3.2 顶部过程摘要

任务刚发出后，过程区顶部只展示“正在思考”，并带蓝色扫光效果，不额外出现大块内容。

第一轮有结果后，顶部改为“已处理 X 秒”，并固定在过程区最前面。它下面有一条横向分隔线，分隔线下面才是所有执行过程。

验收：

- 刚开始只看到“正在思考”。
- “正在思考”有循环扫光，不是静态文字。
- 有第一轮结果后，顶部稳定显示“已处理 X 秒”。
- 顶部摘要不被后续过程顶走。

### 3.3 过程时间线

所有思考、执行、反馈都按顺序追加。

展示节奏是：

`思考或执行过程 → 反馈文案 → 思考或执行过程 → 反馈文案`

旧内容不能被新内容替换。用户视线默认跟随最新输出；如果用户主动往上看历史，则不要强行拉回底部，保留一个回到底部入口。

验收：

- 多轮思考内容不会把前一轮冲掉。
- 多个动作按顺序保留。
- 最新输出默认可见。
- 用户回看历史时不会被自动打断。

### 3.4 命令展示

执行命令默认只展示一行摘要，不展示完整命令和长输出。

用户点展开后，才展示完整内容。展开区域有最大高度，超过高度后只在区域内部滚动，不能把整个过程区无限撑高。

运行中的命令行也带蓝色扫光或轻动效，表示正在处理。

验收：

- 命令默认只有一行。
- 展开后能查看完整命令和输出。
- 长内容在内部滚动，不撑爆页面。
- 出错时仍能清楚看到错误，不被折叠隐藏。

### 3.5 反问面板

反问不再放在执行过程里，而是放在输入框紧挨着的上方，样式参考用户提供的 Codex 截图。

面板结构：

- 顶部：问题标题。
- 右上：上一题、`当前题 / 总题数`、下一题、关闭。
- 中间：候选项列表。
- 底部：填写其他内容、跳过按钮、必要时显示发送或完成按钮。

单选规则：

- 点某个选项后自动进入下一题。
- 如果是最后一题，点选后完成整组反问。

多选规则：

- 可勾选多个选项。
- 点“完成”进入下一题。
- 如果是最后一题，点“完成”完成整组反问。

其他输入规则：

- 用户可在“填写其他内容”里输入自己的答案。
- 有内容时显示发送按钮。
- 发送后进入下一题或完成整组反问。

跳过规则：

- “跳过”只跳过当前题。
- 右上角关闭按钮等同于跳过全部问题。
- 已答过的问题可以返回修改。

验收：

- 面板紧贴输入框上方。
- 多题时显示 `1 / 4` 这类进度。
- 单选自动下一题。
- 多选可勾选并点击完成。
- 支持其他输入和跳过。
- 关闭按钮跳过全部问题。
- 处理完后面板消失，输入框恢复普通补充状态。

### 3.6 步骤提示

输入框上方显示“转圈 + 第 X / X 步”。

鼠标放上去时，出现步骤详情悬窗。每个步骤前面有状态：

- 正在进行
- 还未开始
- 已经完成

每条步骤文案不超过 15 个字。文案优先来自小鲸鱼自己的中间计划或当前动作摘要，不够时再用兜底文案。

验收：

- 运行中能看到当前第几步。
- 鼠标悬停能看到步骤详情。
- 每条文案短，不超过 15 个字。
- 状态清楚，不把未开始误显示成完成。

### 3.7 完成折叠

完成后过程可以折叠，但顶部摘要文案要统一改成：

`思考 X 次，执行了 X 个动作，总耗时 X 分 X 秒`

字号、字体、颜色保持一致，不再出现耗时字号偏小的问题。

验收：

- 完成后默认折叠过程。
- 摘要文案按新格式展示。
- 字号和字体一致。
- 展开后仍能看到完整过程。

### 3.8 颜色统一

普通重点色统一使用小鲸鱼蓝。

需要收敛的地方：

- 文档内容里的普通强调色。
- 过程卡片的普通重点色。
- 链接、选中、进度、扫光效果。

红色只保留给：

- 错误
- 危险
- 拒绝
- 删除等高风险动作

验收：

- 普通强调不再大片出现红色。
- 小鲸鱼蓝成为主强调色。
- 错误和危险仍然醒目。

### 3.9 异常白块

运行命令时，不应出现悬浮在其他过程内容上的小白块。

处理原则：

- 先复现白块出现的场景。
- 找到它来自哪个显示入口。
- 如果只是重复展示，则保留时间线里的命令行，去掉漂浮展示。
- 如果它承担必要状态提醒，则改成时间线内展示，不再悬浮。

验收：

- 执行命令时不会有小白块漂在过程上方。
- 命令状态只出现在正常时间线里。
- 不影响错误、授权、反问这类重要提示。

## 四、改动边界

本轮会影响：

- 输入区：发送、停止、追加、反问面板。
- 过程区：思考摘要、已处理时间、过程追加、命令折叠、完成折叠。
- 步骤提示：底部进度和悬停步骤详情。
- 样式：蓝色色调、红色收敛、动效统一。
- 任务记录：保证过程内容持续保留。

本轮不做：

- 不重做整套页面。
- 不新增复杂产物编辑器。
- 不新增完整富文本输入框。
- 不删除历史旧代码。
- 不把右侧栏重新设计成新产品。
- 不提前做无关能力。

## 五、实施顺序建议

为了降低返工，建议按以下顺序小步推进：

1. 输入区和停止按钮：先把最明显的入口修正。
2. 反问面板：按截图做成输入框上方连续选择面板。
3. 顶部过程摘要：补“正在思考”和“已处理 X 秒”。
4. 过程时间线：确保思考、动作、反馈持续追加。
5. 命令展示：默认一行，展开内部滚动。
6. 步骤提示：补悬停详情和短文案。
7. 完成折叠：改文案和字号。
8. 颜色统一：普通强调改蓝，错误保红。
9. 异常白块：实际复现后定位并去掉漂浮入口。

每一步都要单独验证，通过后再进行下一步。

## 六、风险和处理

### 6.1 当前已有未提交改动

处理：只改本轮必须位置。动手前先查看文件内容，不覆盖已有工作。

### 6.2 反问面板可能和普通输入冲突

处理：反问面板优先处理当前问题；普通输入框仍保留，但不会把普通补充误当成反问答案。

### 6.3 多题反问需要保存中间选择

处理：每题选择先暂存，用户可返回修改；最后统一提交，或按当前任务能力逐题提交。实际采用哪种，以现有任务等待方式为准。

### 6.4 过程追加可能让页面变长

处理：完成后折叠历史过程；命令详情内部滚动；用户主动回看时不强制跳到底部。

### 6.5 步骤总数不一定一开始就知道

处理：有明确总数时显示 `第 X / X 步`；没有明确总数时先显示当前阶段，拿到计划后再补齐总数。

### 6.6 异常白块触发条件未确认

处理：后续实际跑任务复现。复现前不猜根因，不直接删相关显示入口。

## 七、验证方案

完成后至少验证：

1. 发出任务后，运行中空输入显示圆形内方块停止按钮。
2. 运行中输入内容后，按钮变为追加发送。
3. 刚开始只显示“正在思考”并有蓝色扫光。
4. 第一轮结果后，顶部显示“已处理 X 秒”。
5. 多轮过程持续追加，旧内容不消失。
6. 命令默认只显示一行。
7. 命令展开后内容不会撑爆页面。
8. 反问面板出现在输入框上方。
9. 多题反问能按 `1 / 4`、`2 / 4` 连续处理。
10. 单选、多选、其他输入、跳过当前、跳过全部都能正确继续。
11. 步骤提示能显示当前步，并能悬停查看详情。
12. 完成折叠文案为“思考 X 次，执行了 X 个动作，总耗时 X 分 X 秒”。
13. 普通强调色为蓝色，错误仍为红色。
14. 执行命令时不再出现异常悬浮白块。
15. 历史对话恢复后，过程仍可展开查看。
16. 现有发送、附件、文件引用、权限确认不被破坏。

## 八、交付标准

交付时必须说明：

- 实际改了哪些用户可见体验。
- 每一项体验用什么方式验证。
- 是否复现并解决了异常白块。
- 反问面板是否覆盖多题、单选、多选、其他、跳过。
- 是否存在暂缓或无法确认的内容。
