## 本章要回答的真实问题

1. **Q1:Agent 上线了,老板问"表现好不好",除了成功率还能看哪些指标?各自代表什么?** — 场景:5 类维度——任务完成率 / 过程质量(步数·工具调用数)/ 成本(Token·延迟)/ 安全合规 / 用户满意度。不同业务权重完全不同。
2. **Q2:用 AI 评测 AI(LLM-as-Judge),结果靠谱吗?什么条件下能信?** — 场景:阿里大淘宝 91.9% 准确率,但前提是"结构化 Benchmark + 多评审投票 + 人工抽样"三件套缺一不可。
3. **Q3:没历史数据、没评测集,新项目怎么从零建第一套评测体系?** — 场景:真实场景数据打底 → 故障注入补边界 → 模拟系统补长尾。"先评测集再上线"是生产心态。
4. **Q4:A/B 测试在 Agent 场景能直接用吗?有什么特殊挑战?** — 场景:任务多步骤(一次跑 20 步,每步影响结果),样本量和置信区间算法完全不同。AI PM 面试高频考题。
5. **Q5:评测集建完三个月就过时、跟业务脱节,怎么办?** — 场景:"Look at your data"——定期回到真实数据,而非只看评测集指标。"每发现一个新问题就加入评测集"的持续维护机制。

## 章节骨架

1. PM 钩子:你怎么知道这次迭代真让 Agent 变好了?
2. 数据收集方法论:问卷 / 访谈 / 埋点 / 社群信号
3. 离线评估:LLM-as-Judge / 轨迹测试 / 回归集
4. 在线评估:A/B / 灰度 / 置信区间 / 显著性检验
5. 可观测性三支柱:结构化日志 / 执行轨迹 / 指标
6. 关键指标 + 数据→决策完整闭环
