# 单篇知识卡：Agent Loop，Claude Code 的自迭代之道

## 一句话价值

这篇适合用来把“自迭代”讲得更产品化：不是一次生成结果，而是在循环中不断计划、行动、观察、修正。

## 核心观点

- Agent Loop 的价值在于把一次性回答变成持续任务推进。
- 自迭代不是无限循环，而是每一步都要基于观察结果修正下一步。
- 对代码类 Agent 来说，执行、检查、修复、再检查，是最直观的自迭代闭环。
- 第七章可借它解释：自进化比自迭代更进一步，要求把这次循环里的经验沉淀给未来任务。

## 第七章可用弹药

建议用于区分三个词：

| 词 | 大白话 |
|---|---|
| 自迭代 | 这次任务里边做边改 |
| 自进化 | 把这次任务经验留给下次 |
| 自演化 | 长期看，整套系统的能力结构跟着环境变化 |

## 产品经理追问

- 这个系统只是在单次任务内修正，还是会把经验留到未来？
- 每一步观察结果是否会影响下一步？
- 任务结束后有没有复盘和沉淀？
- 循环有没有停止条件，防止无限试错？
