# 单篇知识卡：让 Skill 从执行中生长

## 一句话价值

这篇适合做第七章“Skill 自进化”主案例：把一次任务里的成功路径和踩坑经验，沉淀成下次可调用的办事手册。

## 核心观点

- Skill 不应该只靠人工预写，也可以从真实执行过程中生长出来。
- 任务结束不是终点，系统应回看过程，提取成功步骤、失败修正、人工确认点。
- Skill 自进化至少要包含三段：生成、验证、复用/更新。
- Skill 的价值不只是“写下来”，而是下次遇到类似任务能被正确触发。

## 第七章可用弹药

可用这个类比：

> Memory 像“记住你不吃香菜”，Skill 像“办签证材料该按什么顺序准备”。前者解决知道什么，后者解决会怎么做。

可放入新版大纲第二节：

```text
复杂任务完成
→ 回看过程
→ 提炼步骤和踩坑
→ 写成 Skill
→ 标明触发条件和边界
→ 下次自动调用
→ 新坑继续更新
```

## 产品经理追问

- Skill 是从真实任务沉淀来的，还是凭空写的？
- Skill 的触发条件是什么？
- 有没有验证 Skill 真的减少失败？
- Skill 变多后怎么合并、废弃、回滚？
