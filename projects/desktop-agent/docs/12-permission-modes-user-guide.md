# 小蓝鲸桌面端 · 权限模式用户说明

> 状态：开工前产品视角文档（2026-06-30）
> 性质：把 05 的工程安全规则翻译成用户能理解的产品体验。用户选择模式看本文件，工程判定看 05。
> 参考启发：ZCode 的默认模式、变更前确认、自动编辑、计划模式、完全访问，以及允许/始终允许/拒绝/始终拒绝。

---

## 一、为什么要有权限模式

小蓝鲸能读文件、改文件、执行命令、整理文件夹。能力越强，越需要用户看清它准备做什么。

权限模式的目标不是打断用户，而是让用户在“效率”和“安全”之间自己选择。

---

## 二、五种用户可理解的模式

| 模式 | 用户看到的名字 | 说明 | 适合场景 | 一期 |
|---|---|---|---|---|
| Safe | 变更前确认 | 每次改文件或执行命令前都确认 | 关键代码、重要资料、生产配置 | ✅ |
| Balanced | 默认模式 | 只读自动，普通写入先快照，高风险确认 | 日常开发和办公 | ✅ 默认 |
| Auto Edit | 自动编辑 | 文件编辑自动执行，命令/删除/外发仍确认 | 常规迭代开发 | 二期 |
| Plan | 计划模式 | 先出计划，用户确认后再实施 | 重构、迁移、长任务 | ✅ 一期以“计划确认”形式支持 |
| Full Access | 完全访问 | 尽量自动执行，减少确认 | 已确认上下文可信、希望连续执行 | 二期，且高风险仍确认 |

**映射到 05 工程规则**：
- Safe ≈ 所有写操作都需确认。
- Balanced ≈ low 自动、medium 快照、high 确认。
- Auto Edit/Full Access 是未来模式，不影响一期安全底线。

---

## 三、权限确认弹窗里的决策选项

| 选项 | 用户理解 | 一期 |
|---|---|---|
| 允许 | 只允许这一次 | ✅ |
| 拒绝 | 禁止这一次 | ✅ |
| 始终允许 | 以后同类操作不再问 | 二期 |
| 始终拒绝 | 以后同类操作直接拒绝 | 二期 |
| 允许本会话 | 当前任务里同类操作不再问 | 二期 |
| 始终允许本项目 | 当前项目里同类操作不再问 | 二期 |

一期只做“允许/拒绝”。二期再做记住偏好，避免太早引入复杂白名单管理。

---

## 四、哪些场景一定会确认

- 执行 Shell 命令。
- 安装依赖。
- 删除、覆盖、批量移动文件。
- 修改共享文档或准备外发内容。
- 访问外部网络且可能上传本地内容。
- 任何命中 high/critical 风险的操作。

---

## 五、哪些场景会直接拒绝

这些操作不允许通过“确认”绕过：

- 删除系统根目录或用户主目录。
- 格式化磁盘或写裸设备。
- `curl/wget | sh` 这类远程脚本直接执行。
- 关机、重启、危险 sudo 删除。

这些对应 05 的 BLOCKED 黑名单。

---

## 六、用户提示文案原则

权限确认文案必须说人话，不能只显示工具名。

错误示例：
> tool: shell, args: {command: "npm install"}

正确示例：
> 小蓝鲸想在当前项目里安装依赖：`npm install axios`。这会修改项目依赖文件，可能影响运行环境。是否允许本次执行？

每次确认必须展示：
- 它想做什么。
- 会影响哪些文件/系统。
- 能不能回滚。
- 风险等级。
- 用户可选动作。

---

## 七、计划模式的体验

计划模式不是单独一套 Agent，而是在 L3 长任务前多一个“计划确认”环节。

流程：

```
用户输入复杂任务
→ 小蓝鲸生成计划
→ 用户确认/修改计划
→ 小蓝鲸开始执行
→ 执行中遇高风险仍弹权限确认
```

适合：重构、迁移、整理大量文件、生成长报告。

---

## 八、本文档边界

- 本文档是用户说明，不是代码判定规则。代码以 05 为准。
- 一期只实现 Safe、Balanced、Plan 体验；Auto Edit 和 Full Access 二期。
- “始终允许/始终拒绝/本会话/本项目”二期实现，先写入产品设计，避免未来补丁式扩展。
