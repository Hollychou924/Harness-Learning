# 小蓝鲸桌面端 · 权限规则手册

> 状态：开工前必读契约（2026-06-30）
> 性质：可执行的判定规则，不是示例。AI 执行任何操作前按此判定风险等级和处置方式，禁止自行判断未列操作的风险。
> 核查范围说明：依据已核查的 reference/desktop-claw 的 tools/index.ts（SAFE_TOOLS/BLOCKED_PATTERNS/工作目录边界/超时）、agent-xiaoai/agent.ts（审批回调与拒绝处理），以及小蓝鲸文档第九章、手机端 roadmap 的风险分级。
> 安全原则（对应小蓝鲸第九章）：桌面 Agent 的信任基础不是"模型自觉"，是权限分级、快照回滚、工作目录边界、审计留痕、高风险强确认。权限门独立于模型存在，模型不可绕过。

---

## 一、判定优先级（规则引擎执行顺序）

任何操作按此顺序判定，命中即停：

```
1. 命中 BLOCKED 黑名单 → 直接拒绝，不进审批
2. 在 SAFE 白名单 → 免审批自动执行
3. 按风险等级表判定 → low 自动 / medium 快照后执行 / high 强确认 / critical 禁止自动
4. 未命中以上 → 一律按 high 处理(强确认)
```

**第 4 条是兜底**：遇到没列到的操作，默认最高谨慎，不自行降级。这条防 AI 对未知操作脑补低风险。

---

## 二、BLOCKED 黑名单（直接拒绝，不可审批通过）

Shell 命令命中以下任一模式，直接拒绝返回错误，不进入审批流程：

| 模式 | 正则(参考已验证) | 理由 |
|---|---|---|
| 递归删根 | `\brm\s+(-rf?\|--recursive)\s+[/~]` | 防删系统 |
| 格式化 | `\bmkfs\b` | 破坏磁盘 |
| 写裸设备 | `\bdd\s+.*of=\/dev\/` | 破坏磁盘 |
| fork炸弹 | `:(){ :\|:& };:` | 耗尽资源 |
| 全局权限改 | `\bchmod\s+(-R\s+)?777\s+\/` | 安全风险 |
| 关机重启 | `\bshutdown\b` `\breboot\b` | 影响系统 |
| 写磁盘设备 | `>\s*\/dev\/sd` | 破坏磁盘 |
| 管道执行远程脚本 | `\bcurl\b.*\|\s*(ba)?sh` `\bwget\b.*\|\s*(ba)?sh` | 执行任意远程代码 |
| sudo删 | `\bsudo\s+rm\s+` | 提权删除 |
| python提权破坏 | `\bpython[23]?\s+-c\s+.*import\s+os.*system` | 任意代码执行 |

**AI 实现要求**：这组正则作为常量数组，shell 工具 execute 入口先逐一 test，命中任一即返回 `Blocked: 危险命令已拒绝`。

## 三、SAFE 白名单（免审批自动执行）

以下只读操作免审批，依据参考已验证清单：

- 文件只读：`read_file` `list_files`
- 记忆只读：`memory_search` `memory_get`
- 浏览器只读：`browser_snapshot` `browser_take_screenshot` `browser_wait_for`
- 定时任务只读：`list_scheduled_tasks` `get_scheduled_task`
- MCP 只读查询类：`list_targets` `get_page_content` `get_cookies` `get_network_log` `get_console_logs` 及各 IM/文档的 list/search/get 类

**判定规则**：工具名在白名单集 → 免审批。其余一律需审批。

## 四、风险等级表（非黑非白操作按此）

| 等级 | 操作类型 | 处置 | 示例 |
|---|---|---|---|
| low | 只读/查询/草稿生成 | 自动执行 | 总结文档、搜索、生成草稿 |
| medium | 改文件/生成文件/重命名 | 建快照后执行,可回滚 | 改代码、新建文件、重命名 |
| high | 删除/覆盖/批量移动/Shell写/安装依赖 | 必须用户确认,展示风险点 | 删文件、git push、npm install |
| critical | 覆盖系统文件/外发数据/改共享文档 | 禁止自动,必须确认且不可批量 | 覆盖 ~/.ssh、发送邮件、改共享飞书文档 |

**关键规则**：
- medium 必须先建 snapshot（snapshot_store），失败可回滚。
- high 必须弹确认，展示"要做什么+影响什么+能否回滚"。
- critical 即使确认也一次一个，禁止批量。

## 五、权限模式（对应小蓝鲸第 9.1 节）

| 模式 | 策略 | 一期 |
|---|---|---|
| Safe Mode | 只读自动,所有写操作确认,高风险强确认 | ✅ |
| Balanced Mode | low 自动,medium 快照后执行,high 确认 | ✅ |
| Auto Mode | 授权目录内自动,高风险仍确认,白名单/禁止目录可配 | 二期 |

一期实现 Safe + Balanced 两种。默认 Balanced。

---

## 六、工作目录边界（依据参考已验证）

- Shell 执行目录必须在用户主目录(`os.homedir()`)下，超出直接拒。
- 写文件操作限定在 `user_workspace_dir`（AgentConfig 配置）内，超出拒。
- 这条是物理边界，独立于风险等级——即使 low 风险，路径越界也拒。

---

## 七、审批流程（依据参考已验证 agent.ts）

```
工具调用 → 判定需审批(high/critical或非白名单写)
  → 弹确认(展示:工具名/参数/风险点/影响文件)
  → 用户批准 → 执行
  → 用户拒绝 → 返回拒绝结果给模型
```

**拒绝后的处理（参考已验证，必须照做）**：
返回给模型的消息格式：`[用户拒绝] 用户未授权执行 "{toolName}"，请尊重用户意愿，不要重复尝试相同操作。可以改用其他方式或询问用户。`

这条防止模型被拒后反复重试同一操作（死循环）。

---

## 八、快照回滚机制（对应小蓝鲸第 9.2 节）

| 时机 | 动作 |
|---|---|
| medium/high 写操作前 | 建 snapshot（受影响文件副本+元数据入 snapshot_store） |
| 任务失败/用户要求回滚 | 从 snapshot 还原受影响文件 |
| 任务 COMPLETED 且用户确认 | 可选保留或清理 snapshot |

**约束**：
- 回滚是文件级（还原受影响文件），不是 git 级（不强依赖 git）。
- Code 场景可优先用 git stash 作为快照手段（项目已在 git 时）。
- 回滚失败必须明确报错，不能静默。

---

## 九、审计留痕

所有操作的判定结果和审批结果必须写入 event_log（见 04 文档 1.2）：
- event_type: APPROVAL_REQUESTED
- tool_name / tool_args / approved（0或1）
- risk_level（写入 tool_args 或单独字段）

这条保证任何操作可追溯、可复盘——对应架构反思"不过度简化"（安全审计不能省）。

## 十、本文档边界

- 不定 UI 确认弹窗的样式（那是 UI 文档）。
- Auto Mode 的白名单/禁止目录配置规则二期补。
- 跨端工具调用的权限（对端请求本地执行时的审批）待跨端调研后补——一期不实现跨端，此项暂缺但显式标注。

## 十一、与用户版权限模式的映射（2026-06-30 补充）

用户能理解的权限模式见 `12-permission-modes-user-guide.md`。工程规则映射如下：

| 用户模式 | 工程策略 | 一期 |
|---|---|---|
| 变更前确认（Safe） | 所有写操作和命令都需确认 | ✅ |
| 默认模式（Balanced） | low 自动、medium 快照、high 确认 | ✅ 默认 |
| 自动编辑（Auto Edit） | 文件编辑自动，命令/删除/外发仍确认 | 二期 |
| 计划模式（Plan） | L3 前先生成计划并等用户确认 | ✅ 以计划确认形式支持 |
| 完全访问（Full Access） | 尽量自动执行，但 high/critical 仍确认 | 二期 |

一期审批选项仅实现：允许 / 拒绝。始终允许、始终拒绝、允许本会话、始终允许本项目二期实现，避免提前引入复杂白名单管理。
