# 小蓝鲸桌面 Agent · 达到业内领先的拆解计划

> 读者：产品 / 研发负责人  
> 日期：2026-07-12  
> 前提：架构已定为 L0←L1←L2←L3（见 `17` / `18`）  
> 目标：把「设计第一梯队」变成「产品可宣称的业内领先」  
> **执行状态（2026-07-12）：M0 进行中并已连带开工 M1 骨架** — 详见 `20-m0-execution-baseline.md`、`src/agent/src/evolution/`、`eval/golden-t1/`

---

## 〇、先定义：什么叫「业内领先」

不能拿一句「我们有自进化」对外。领先 = **两根柱子同时过线**：

| 支柱 | 对标谁 | 领先含义（可验收） |
|------|--------|-------------------|
| **A. 执行 Harness（L0）** | Codex / Claude Code / 一线桌面 Agent | 长程任务稳、可中断、不空转、上下文能撑住；人机卡点不拖垮体验 |
| **B. 自进化 Loop OS（L1–L2，L3 预留）** | Meta-Harness 思路 + 车控 BVT 实战密度 | **可审计地越用越强**：经验复用、环外验收、元反思、离线/在线拆分——多数上线产品尚未产品化到这一层 |

**判定公式：**

```text
业内领先 = A 达到「一线可用」 ∩ B 达到「可对外讲清且指标好看」
只做 A → 好用的跟跑者
只做 B → 论文/垂直很强，通用桌面仍弱
```

**明确不抢的领先：** 基础模型本身、全自动改安全策略、开放域 100% 一次成功。

## 一、现状基准（起跑线）

| 维度 | 现状 | 距领先 |
|------|------|--------|
| L0 执行 | ReAct + 计划/提问/审批/续跑 + Turn/Item | 缺：压缩、防空转、真中断、子 Agent、长任务治理 |
| L1 经验 | 基本无 | 整层缺失 |
| Verifier / 黄金集 | 无产品级环外尺子 | 整层缺失 |
| 反思治理 | 无（且需防假复查） | 缺失 |
| 离线/在线 | 未拆 | 缺失 |
| L2 流程进化 / 元反思 | 无 | 缺失 |
| 垂直旁证 | 车控 BVT Loop 已很强 | **可迁移机制，不直接等于桌面领先** |

---

## 二、总路线：四段里程碑

```text
M0  定义与样板    →  能测「有没有变强」
M1  进化最小闭环  →  B 支柱立住（设计变产品）
M2  执行一线化    →  A 支柱立住（对标一线 Harness）
M3  流程进化+元反思 →  A∩B 可对外宣称领先
M4  （可选）跨端/权重/L3 → 巩固壁垒
```

建议节奏（可按人力伸缩，**顺序不要倒**）：

| 里程碑 | 建议周期 | 一句话出口 |
|--------|----------|------------|
| **M0** | 2 周 | 尺子 + 样板任务族 + 领先指标看板 |
| **M1** | 4–6 周 | 同类任务明显少踩坑；复查不再假忙 |
| **M2** | 6–8 周 | 长程 Code 任务完成率与稳定性进入一线区间 |
| **M3** | 4–6 周 | 流程可进化可回滚；元反思按期；对外口径成立 |
| **M4** | 按需 | 手机共用经验；微调通道；L3 仍可选 |

## 三、拆解计划（按里程碑）

### M0｜立尺子（没有尺子就没有「领先」）

**目标：** 全体对齐「领先长什么样」，并可周更看数。

| # | 事项 | 必要性 | 出口标准 |
|---|------|--------|----------|
| M0.1 | 选定 **样板任务族 T1**：Code「小改动 + 跑测试/编译」 | 客观信号最硬，先打穿 | 有 20+ 可重复用例 |
| M0.2 | 定义 **黄金集 Golden-T1**（已通过基线） | 防修 A 坏 B | CI/脚本一键回跑 |
| M0.3 | 定义北极星与过程指标（见第四节） | 否则无法证明领先 | 看板每周可更新 |
| M0.4 | Evidence 字段冻结（Turn/Item + 接管点） | L1/L2 的粮食 | 失败任务可回放卡点 |
| M0.5 | 写清对外/对内口径（设计领先 ≠ 已领先） | 防过度承诺 | 一页话术评审通过 |

**M0 完成 = 可以开始冲 M1/M2，而不是凭感觉迭代。**

---

### M1｜B 支柱：自进化最小闭环（先做到「会积累」）

对应产品方案 P0：证据 → Verifier → L1 → 反思治理。

| # | 事项 | 做什么 | 出口标准 |
|---|------|--------|----------|
| M1.1 | 环外 Verifier v1 | 目标达成 / 回归 / 工具成功 / 安全 / 人机负担 / 成本 | T1 任务自动出分 |
| M1.2 | failure_cases + gap_pattern | 可执行经验，不是日记 | ≥30 条有效案例；pattern≥10 |
| M1.3 | Tactic 注入 | 下次同类任务带 Top-K 策略 | 注入命中可统计 |
| M1.4 | ReflectionGovernor | 失败才复查；有预算；无证据不开 | 简单任务复查次数↓≥40%，成功率不降 |
| M1.5 | 短失败环 | 失败后 ≤3 轮按 pattern 修再测 | 同失败二次出现率↓ |
| M1.6 | 离线批跑雏形 | 夜间对 T1 跑进化，不进用户等待 | 用户会话默认不跑长进化 |

**M1 退出门禁（B 支柱「立住」）：**

- T1 **一次交付成功率** 相对 M0 基线 **+15pt** 或达到约定绝对值（立项时钉死）  
- 同族 **平均重试次数 ↓≥30%**  
- 黄金集 **零退化** 作为发布门禁  
- 产研能打开「经验账本」并指出 3 个已被复用的坑  

**此时对外可说：**「我们开始具备可审计的自进化，而不仅是多轮反思。」  
**仍不可说：**「执行体验已全面领先 Codex/Claude Code。」

---

### M2｜A 支柱：执行 Harness 一线化（先做到「长任务敢交」）

没有 A，B 的领先会被「动不动做不完」抵消。

| # | 事项 | 对标能力 | 出口标准 |
|---|------|----------|----------|
| M2.1 | 上下文压缩 + 压缩循环检测 | Codex compact / 长会话 | 长任务完成率↑；压缩后目标不丢 |
| M2.2 | LoopGuard 防空转 | 重复工具/乒乓检测 | 空转任务可自动停并提示拆分 |
| M2.3 | 真暂停 / 取消 | 控制感 | 审批等待中可取消；无死锁 |
| M2.4 | 续跑 / 拆分体验打磨 | 已有 continuation，做透 | 用户理解「继续/停止/拆分」 |
| M2.5 | 动作合并与完成折叠（交互） | 13 号 PRD | 长任务不刷屏；结果感压过日志感 |
| M2.6 | （可选）子 Agent 试点 | Claude Task | 仅 T1 内 1 类委派跑通 |

**M2 退出门禁（A 支柱「立住」）：**

- 定义「长程 Code 套件」：步数/文件数/需验证的真实任务 N 个  
- **完成率、中位耗时、用户强制停止率** 达到内部对标线（建议：对标你们选定的竞品集测，公开竞品用同套件盲测）  
- 压缩触发后 **目标/约束保留率 100%**（抽检）  
- 空转导致的死任务 **≈0**（有熔断）  

**此时对外可说：**「长程执行进入一线桌面 Agent 区间。」  
**与竞品差：** 若子 Agent/Smart Approval 未做完，口径写「核心路径一线」，不写「全面超越」。

---

### M3｜A∩B：流程进化 + 元反思（可宣称「业内领先范式落地」）

| # | 事项 | 做什么 | 出口标准 |
|---|------|--------|----------|
| M3.1 | L2 受限 Harness 进化 | 只搜：Skill 触发、复查开关、压缩阈值、审批松紧、提示策略 | 候选→回测→人工发布→可回滚 |
| M3.2 | 安全红线锁定 | 黑名单等不进自动搜索面 | 审计通过 |
| M3.3 | 元反思制度 | 每 5 个版本强制；必须产出规则变更或「无需变更」证据 | 连续 2 个周期按期完成 |
| M3.4 | Work 任务族 T2 试点 | 报告类：产物结构/来源/用户确认为 Verifier | T2 一次成功率有基线并改善 |
| M3.5 | 领先叙事与评测报告 | 对内评测报告 + 对外一页纸 | 法务/产研对齐话术 |

**M3 退出门禁（「业内领先」内部立项标准）：**

同时满足：

1. **A**：M2 门禁保持不回退  
2. **B**：M1 门禁保持；且 T1 一次成功率再提升一档或稳定在高位  
3. **范式**：有≥1 次 Harness 版本发布因元反思/L2 产生，且 Golden 通过  
4. **可审计**：任意季度可回答「变强证据、回滚记录、红线未动」  
5. **对比**：同套件下 vs 选定竞品，**自进化相关指标明显领先**（经验复用率、重复坑率、复查 ROI）；执行指标 **不显著落后**

**此时对外建议口径：**

> 在通用桌面 Agent 中，我们把「执行环 + 经验环 + 流程进化 + 元反思」做成可验收系统；  
> 在样板任务族上，重复踩坑与无效复查优于常见「只靠多轮自我反省」的产品；  
> 长程执行达到一线可用性。

### M4｜巩固壁垒（可选，不挡「首次宣称」）

| # | 事项 | 价值 |
|---|------|------|
| M4.1 | 桌面 ↔ 手机同一 Experience schema | 端云一体经验 |
| M4.2 | 高置信轨迹 → 微调通道（人工 promote） | 能力进模型 |

| M4.4 | Smart Approval（规则+可选分类器） | 减负不减安全 |

## 四、指标拆解（用来证明领先）

### 4.1 北极星

| 指标 | 定义 | 领先方向 |
|------|------|----------|
| **T1 一次交付成功率** | 无失败重开会话即达成验收 | ↑ 且稳 |

### 4.2 A 支柱（执行）

| 指标 | 领先方向 |
|------|----------|
| 长程套件完成率 | ↑ |
| 用户强制停止率 | ↓ |
| 空转熔断次数（应熔断却没熔断） | → 0 |
| 压缩后关键约束丢失率 | → 0 |

### 4.3 B 支柱（自进化）

| 指标 | 领先方向 |
|------|----------|
| 同族平均重试次数 | ↓ |
| 经验命中后的成功率提升 | ↑ |
| Critic token 占比 | ↓ 或持平且成功率↑ |
| 黄金集回归失败 | 发布时 = 0 |
| 元反思按期完成率 | = 100% |

### 4.4 对比评测（建议固定「对手盘」）

内部每季度用同一套 T1/长程套件盲测：

- 至少 1 个一线 coding CLI/桌面（执行对标）  
- 至少记录「有无经验复用 / 有无环外验收 / 有无元反思」——多数竞品此处为弱或无  

**领先声明必须带套件版本号与日期**，避免空话。

## 五、依赖与反顺序（防止假领先）

| 禁止 | 原因 |
|------|------|
| 先上 L3 / 自动改全量 Harness 代码 | 无尺子会系统性改坏 |
| 先上微调再补 Verifier | 脏数据进权重 |
| 未做 M1 就宣传「自进化领先」 | 名不副实 |
| 未做 M2 就宣传「全面超越 Claude/Codex」 | 执行柱子倒了 |
| 用户会话默认长进化 | 通用产品体验自杀 |
| 进化环可改安全黑名单 | 信任与合规崩盘 |

**强制顺序：M0 → M1 与 M2 可部分并行，但 M3 必须 M1+M2 门禁都过。**

建议并行方式：

```text
     M0
      │
      ├──────────────┬──────────────┐
      ▼              ▼              │
     M1 (B)         M2 (A)          │ 共享 Evidence / Golden
      │              │              │
      └──────┬───────┘              │
             ▼                      │
            M3 (A∩B) ◄──────────────┘
             │
             ▼
            M4
```

## 六、组织与样板

| 角色 | 职责 |
|------|------|
| 产品 | 任务族定义、口径、指标、样板优先级 |
| Agent 引擎 | L0/L1/Governor/压缩/LoopGuard |
| 评测 | Verifier、Golden、季度对比套件 |

## 八、一页「冲刺到领先」检查表

- [ ] **M0** 尺子与 T1 黄金集就绪  
- [ ] **M1** 一次成功率↑、重试↓、复查降噪、经验可打开  
- [ ] **M2** 长程完成率一线、空转可熔断、可取消  
- [ ] **M3** Harness 版本可回滚、元反思按期、对比报告出具  
- [ ] 口径：只宣称门禁已过的范围  
- [ ] 红线：安全策略不在自动进化面

## 九、结语

- **领先不是功能清单最长**，而是 **A（一线执行）∩ B（可审计自进化）**。  
- 你们的差异化主战场在 **B**（多层 Loop OS）；**A** 是入场券，必须过。  
- 车控 BVT 证明 B 的路能跑通；本计划是把它迁到**通用桌面 Agent** 并补齐 A 的拆解图。  

**按 M0→M3 门禁走完，才具备「业内领先」的对内立项与对外谨慎宣称资格。**
