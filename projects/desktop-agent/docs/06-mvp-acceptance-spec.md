# 小蓝鲸桌面端 · MVP 验收规格书

> 状态：开工前必读契约（2026-06-30）
> 性质：定义一期"做完了"的客观标准。每个模块有可勾选的验收条目，AI 不得声称完成除非条目全绿。
> 核查范围说明：依据小蓝鲸文档第十四章（MVP 范围）、你锁定的第一个真实 case（做网页/定位 bug/整理桌面）、04 数据契约、05 权限规则。
> 一期策略（对应小蓝鲸第十四章）：Code 深、Work 浅；Mac 优先；云端 API。

| 模块 | 一期目标 | 验收章 |
|---|---|---|

## 二、Unified Runtime 验收

- [ ] Preflight 能判定 L0/L1/L2/L3 四级，简单问题短路返回不进长任务
- [ ] 7 步闭环完整跑通：Goal Contract→Plan→Capability Binding→Tool Exec→Validation→Artifact Review→State 沉淀
- [ ] ReAct 引擎有 maxIterations 上限，到顶兜底收尾不卡死
- [ ] 工具执行后 LLM 空回复能重试一次
- [ ] 工具后报错能合成收尾，不让对话悬空
- [ ] 任务状态机按 04 文档第二章转移，终态正确
- [ ] Agent 引擎作为独立子进程运行，崩溃不影响 UI
- [ ] Agent 引擎不 import 任何 Electron API

**验证方式**：单测覆盖 Preflight 分级、状态机转移、ReAct 兜底；集成测试跑一个 L3 任务全流程。

## 三、Code Tab 验收（你定的 case：定位 bug）

第一个真实 case：**读一个本地项目 → 定位一个 bug → 改 → 跑测试 → 出 Diff**

- [ ] 能选择本地项目目录并索引（workspace_index 写入）
- [ ] 代码问答：问项目结构/某函数用途，能基于索引回答
- [ ] 代码搜索：按符号/文本搜到位置
- [ ] 小范围修改：改 1-3 个文件，生成 Diff
- [ ] 修改前建 snapshot，可回滚
- [ ] 能运行测试命令并展示结果
- [ ] 交付物含：修改摘要、Diff、测试结果、风险说明、下一步建议
- [ ] Shell 写操作经审批，命中 BLOCKED 拒绝
- [ ] **端到端验收 case**：在一个有测试的真实项目里，给一个可定位的 bug 描述，Agent 走完 7 步，产出可 apply 的 Diff 且测试通过

**"小范围修改"边界**：一次不超过 3 个文件、单文件改动不超过 100 行。超出需拆分或提示用户。这条防止 AI 越改越大。

## 四、Work Tab 验收（你定的 case：整理资料/做网页报告）

第一个真实 case：**读 3 个网页 → 整理成带来源的报告**

- [ ] 能读取本地文件（MD/PDF/文本）和网页 URL
- [ ] 资料总结：对单个文档/网页生成摘要
- [ ] Markdown 报告草稿：多资料整合成带结构的报告
- [ ] 来源追溯：报告里每条结论能指回来源链接/文件
- [ ] Excel 简单分析：读表结构、基础统计、生成结论
- [ ] 导出：报告/表格可导出为文件
- [ ] 写入默认生成草稿或副本，不覆盖原文件
- [ ] **端到端验收 case**：给 3 个网页 URL，Agent 走完 7 步，产出带来源的 Markdown 报告，来源链接可点击且内容属实

## 九、整体端到端验收（一期完成判据）

以下 3 个 case 全部跑通，才算一期完成：

1. **Code-定位bug**：真实项目+真实bug描述 → 可apply的Diff + 测试通过
2. **Work-整理报告**：3个网页 → 带来源的Markdown报告
3. **整理桌面**：指定文件夹 → 预览整理规则 → 批量移动/重命名 + 可回滚

每个 case 的完成必须：当场执行 + 看完整输出 + 用实际结果说明（对应全局"证据先行"规则）。

## 十、不验收的内容（明确排除）

- 跨端（一期预留接口位，不验收功能）
- PPT 自动生成（二期）
- 多 Agent 协作（二期）
- 自我进化（二期）
- Auto Mode（二期）
- Windows/Linux 打包（一期 Mac 优先，另两端能构建即算，不深度验收）

这些不验收=一期不做，避免范围蔓延。

## 十一、功能成熟度补充（2026-06-30）

为避免后续 AI 把二期功能提前做进一期，成熟度以 `10-product-feature-map.md` 为准：

| 分类 | 一期必须验收 | 二期再做 | 暂缓 |
|---|---|---|---|
| 权限模式 | Safe / Balanced / Plan确认 | Auto Edit / Full Access / 记住偏好 |  |
| Work 能力 | 网页/文档总结、Markdown报告、Excel简单分析 | PPT、邮件、日历、自动化 |  |
| 跨端 | 仅预留接口位 | 手机控制PC（待调研账号绑定方案） |  |
| 模型 | 云端API + 字节embedding | 更多模型策略优化 | 本地模型 |
| 安全 | 黑名单/白名单/快照/审批/审计 | Auto Mode 白名单管理 | 绕过确认 |

一期验收时，不允许把二期功能作为必须项，也不允许因为二期没做而判定一期不完成。
