# 从其他开发工具导入资料

## 1. 目标与首批范围

设置中提供“从其他开发工具导入”入口。首批支持本机的 Codex、Claude Code 和 Cursor。

- Codex：桌面版、命令行版和编辑器入口共用同一批本机资料，产品中只显示一个 Codex 来源。
- Claude Code：仅包含命令行版，不包含 Claude Desktop 和网页端。
- Cursor：包含桌面版；经过样本验证的保存版本可迁移项目、标题和可识别正文，但附件、操作过程和原运行现场不能完整还原，因此标记为“正文可用但部分缺失”。未知版本只展示能够确认的资料。

产品承诺为：迁移本机能够识别、能够读取且通过兼容验证的资料。已删除、仅在网上、位于远程电脑、损坏或无权读取的资料不承诺迁移。

## 2. 用户流程

### 2.1 自动提醒

1. 仅在本机实际发现可迁移资料时提醒。
2. 首次使用流程中的导入项默认收起，不阻断开始使用。
3. 用户跳过后，在用户空闲且没有任务运行时补提醒一次，提供“立即导入”“7 天后提醒”“不再提醒”。
4. “不再提醒”按来源保存；另有关闭全部导入提醒的总开关。
5. 软件升级本身不触发提醒。新增来源且本机确有资料时，可以仅提醒该新来源。

### 2.2 设置入口

设置中永久保留“导入资料”。首次执行显示“开始导入”，之后显示“检查新增资料”。

每次检查只处理新增和已变化的资料，不重复创建未变化内容。来源中删除的内容不会同步删除小蓝鲸内容。

### 2.3 选择与预览

1. 只展示本机检测到的来源，默认全部选中，用户可以取消，但至少保留一个。
2. 每个来源分别展示发现的项目数、正文完整可读数、正文可用但过程缺失数、整份无法读取数及原因。
3. 每个来源内可选择资料类别：
   - 项目与对话默认选中。
   - 项目说明与记忆默认选中，但按来源分别保留，不直接改变现有规则。
   - 扩展与自动设置（能力扩展、外部连接、自动动作和子助手）默认不启用，只导入清单并逐项确认。
   - 附件单独展示数量和预计空间。
4. 预览中展示准备自动合并的项目，证据不足时保持分开，不按名称强行合并。

### 2.4 完成结果

结果页展示新增、更新、跳过重复、只能查看、失败的数量和原因，并提供“撤回本次导入”。

撤回只移除本批新增内容。用户导入后修改或继续产生的内容必须保留；这类内容与导入来源解除关联，不随批次删除。导入时复用的小蓝鲸原有项目不能被撤回删除。

## 3. 资料处理规则

### 3.1 项目

- 优先使用来源明确保存的项目名称；没有时使用文件夹名，并标为自动识别。
- Codex、Claude Code、Cursor 和小蓝鲸原有项目只要指向同一个本机文件夹，就展示为一个项目；同名但位于不同文件夹的项目保持分开。
- 合并时优先保留小蓝鲸原有项目的名称、图标、置顶和排序，所有来源对话迁入该项目；当前正在查看的项目也同步切换到保留项目。
- 自动合并需要多个证据一致，例如真实位置、代码仓库身份和来源记录。项目移动、软链接、大小写差异、多目录工作区和远程位置都要单独处理。
- 无法确定时宁可保留两个项目，由用户手动合并。

### 3.2 对话

- 来源、项目、对话和消息均保存稳定的来源身份与内容指纹，禁止按标题去重。
- 再次检查逐条核对。来源可能重写、压缩、分叉、归档或恢复，不能按“上次第几条”简单追加。
- 用户在小蓝鲸中改过的标题、分类和备注不覆盖。
- 同一对话在两边都继续时，不强行拼接；保留小蓝鲸内容，将来源变化显示为来源更新或新分支。
- 导入对话属于历史资料。继续对话时开启新的后续内容，不宣称恢复当时的模型、工具和文件现场。

### 3.3 项目说明、记忆和扩展

- AGENTS.md、CLAUDE.md、Cursor Rules 等按来源、适用位置和原先顺序分别保留，不能粗暴合成一份。
- 同名记忆和能力扩展不覆盖，先展示差异。Codex 自动生成的历史摘要单独归类，不能冒充用户主动维护的长期记忆。
- 能力扩展、外部连接、自动动作、子助手和命令导入后默认关闭。
- 登录状态、授权信息、密码和密钥不迁移。敏感内容检查覆盖环境变量、网址、命令参数、辅助脚本和说明文本，不能只按字段名过滤。

### 3.4 隐私和稳定性

- 检测和转换全部在本机完成，不把对话、附件、项目说明或记忆发送到外部服务。
- 检查记录只保存数量、状态和脱敏原因，不保存完整对话、密码或完整私人路径。
- 读取前后检查来源是否仍在变化；正在写入的资料延后处理，不能把半截内容标为成功。
- 导入前检查可用空间、读取权限和附件状态。附件失败不影响对话正文。

## 4. 可靠保存

新增统一导入账本，至少记录：来源身份、来源版本、项目身份、对话身份、消息指纹、目标身份、批次、状态、失败原因和检查时间。

每批导入遵循以下顺序：

1. 建立只读资料快照并完成预检。
2. 写入临时批次，不修改当前可见清单。
3. 校验项目、对话正文和账本能够互相对应。
4. 一次性公布本批结果。
5. 中断后恢复未完成批次，或完整撤销临时内容。

同一时间只允许一个导入任务。设置入口和提醒入口同时触发时，后一个入口显示当前进度。

## 5. 自动验证与上线门槛

必须自动覆盖：重复导入、增量更新、两边同时继续、来源重写、项目移动、同名项目、未知 Cursor 版本、来源正在写入、中断恢复、部分失败、空间不足、权限不足、附件缺失、敏感信息拦截、扩展默认关闭、整批撤回和超大数据。

上线前还必须用 Codex、Claude Code、Cursor 的真实本机样本分别完成：首次导入、再次检查、来源新增内容、撤回、撤回后再次导入。

以下条件全部满足才可宣称对应版本“完整兼容”：

- 项目数和可读取对话数与来源样本核对一致。
- 对话角色、顺序、时间、分支和归档状态没有串线。
- 连续导入三次不产生重复。
- 任意阶段中断不会损坏小蓝鲸已有资料。
- 导入的可执行内容均保持关闭，登录与密钥没有迁移。
- 整批撤回不删除用户后来创建或修改的内容。

Windows、Linux、远程开发环境和未验证的历史版本在取得真实样本并通过同一套检查前，不标记为完整兼容。
