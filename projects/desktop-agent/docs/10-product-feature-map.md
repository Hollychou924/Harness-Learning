## 一、小蓝鲸是什么

小蓝鲸不是 Chat，而是一个能在本地工作区执行长程复杂任务的桌面 Agent。

它做三件事：

1. **理解你的工作材料**：代码项目、网页、文档、表格、文件夹。
2. **执行多步骤任务**：修 bug、写功能、整理资料、生成报告、整理文件夹。
3. **交付可检查结果**：Diff、测试结果、报告草稿、来源证据、文件整理预览。

## 二、两大工作台

| 工作台 | 适合谁 | 能做什么 | 一期深度 |
|---|---|---|---|
| Code | 开发者 | 项目理解、代码问答、搜索、修 bug、小功能、Diff、测试、回滚 | 深 |
| Work | 知识工作者 | 网页总结、资料整理、报告草稿、Excel 简单分析、文件导出、来源追溯 | 浅但可用 |

顶部显性展示 Code / Work 两个 Tab。用户不用猜“我该怎么问”，先选工作台。

## 四、典型任务

### Code 场景

- 解释一个陌生项目结构。
- 搜索某个函数在哪里被调用。
- 定位接口偶发 500 的根因。
- 小范围修改 1-3 个文件并跑测试。
- 生成 Diff、测试结果和提交说明。

### Work 场景

- 总结当前文档。
- 把 3 个网页整理成带来源的报告。
- 分析 Excel 并输出结论。
- 整理文件夹：先预览规则，再批量移动/重命名。
- 导出 Markdown 报告。

## 五、用户心智：小蓝鲸和普通 AI 对话的区别

| 普通 AI 对话 | 小蓝鲸 |
|---|---|
| 主要给建议 | 能执行任务 |
| 用户手动操作文件 | Agent 操作文件，但高风险需确认 |
| 单轮问答为主 | 多步骤任务闭环 |
| 输出一段文字 | 输出可检查产物 |
| 做错难回滚 | 修改前建快照，可回滚 |
| 用户看不见过程 | 展示步骤、工具、权限、校验 |
