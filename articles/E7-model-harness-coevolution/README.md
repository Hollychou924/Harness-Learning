# E7｜自进化篇：怎么让 Agent 越用越聪明

> 状态：✍️ 2026-06-15 根据 23 篇微信 + Meta-Harness / Harness evolution 论文重规划
> JD 锚点：与研究员协作 / 训练数据-Harness 闭环 / Eval 飞轮 / Agent 长期迭代
> 定位：接 E6。前 6 章讲“怎么把 Agent 搭起来、跑稳、管住”；E7 要回答“上线后它怎么变强，而不是越用越脏”。

## 本章新的核心判断

原稿方向没错，但偏浅：讲了“错误固化、数据飞轮、治理”，但缺少读者能抓住的主线、硬案例和产品检查表。

新版第七章建议用一句话打穿：

> **很多 Agent 用了一万次也不会变聪明，因为它只是在重复执行，没有把失败变成可复用资产。真正的自进化，是把每一次失败、接管和复盘，变成下次能自动用上的规则、技能、评测题和训练素材。**

## 本章要回答的真实问题

1. **为什么很多 Agent 不会越用越聪明？**  
   因为系统只保存了“最后答案”，没有保存“过程证据、失败原因、人工接管点、验证结果”。没有这些，下一次只能重犯。

2. **一次失败怎么变成下次不再犯？**  
   不是写一句“以后注意”，而是走完整闭环：看见失败 → 归因 → 沉淀到正确层 → 做成评测题 → 验证通过 → 灰度进入正式能力库。

3. **Skill 自进化到底比普通记忆强在哪里？**  
   记忆更像“知道一件事”，Skill 更像“会办一件事”。自进化要把任务里的成功路径、踩坑、人工确认，沉淀成下次可调用的办事手册。

4. **谁负责写经验，谁负责用经验？**  
   最新论文把自进化拆成两种能力：写经验的能力、用经验的能力。很多系统失败不是因为经验没写好，而是下次任务时根本没找到、没调用、没遵守。

5. **Harness 怎么反哺模型训练？**  
   短期先改 Harness：规则、技能、流程、工具选择更好；长期再把高质量过程数据变成训练和评测资产。第七章要把“外部经验沉淀”和“内部模型训练”讲成双飞轮。

6. **自进化怎么不把自己改坏？**  
   Agent 可以提出改进建议，但不能绕过验证直接改正式能力。高风险改动必须有人审、可灰度、可回滚。

## 本章写作红线

- 不写成“AI 自己进化”的玄学故事；所有进化都要落到“记录、复盘、沉淀、验证、发布、复用”。
- 不把 Memory 和 Skill 混为一谈：Memory 解决“记得什么”，Skill 解决“会怎么做”。
- 不只讲生成 Skill，还要讲“下次能不能正确调用 Skill”。
- 不只讲变强，也要讲变脏、过拟合、权限失控。
- 不把模型训练说得太轻松：数据授权、隐私、训练评测隔离必须讲清楚。
